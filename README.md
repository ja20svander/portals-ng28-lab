# portals-ng28-lab

本仓库用于存档与维护多个独立的 HTML 页面。这些页面彼此独立，不针对任何特定域名或网站，仅作为前端静态资源归档使用。

## 项目简介

`portals-ng28-lab` 是一个实验性仓库，主要用于存放各类自包含的 HTML 页面。每个页面均为独立文件，附带必要的样式与脚本，不依赖外部服务或后端。

## 目录说明

```
portals-ng28-lab/
├── index.html          # 仓库主页（可选）
├── pages/              # 页面归档目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 共享资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 页面文件，文件名以 `page-` 开头并编号，便于管理。
- `assets/`：存放页面可能共享的样式、脚本或图像资源，避免在每个页面中重复引用。

## 页面归档说明

- 所有页面均为纯前端实现，不包含后端逻辑。
- 页面间无依赖关系，可直接通过浏览器打开运行。
- 每个页面均包含完整的 HTML 结构，部分页面可能引用 `assets/` 下的资源。
- 归档内容仅供学习、测试或参考用途。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/portals-ng28-lab.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看。

## 维护说明

- 本仓库不定期更新，新增或修改页面时请保持命名规范。
- 建议在 `pages/` 目录下添加页面时，同步更新本 README 文件中的目录结构（如适用）。
- 如需引用外部资源，请确保资源已放入 `assets/` 目录，避免使用外链。
